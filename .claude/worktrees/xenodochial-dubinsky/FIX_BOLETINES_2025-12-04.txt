## 📋 PROBLEMA IDENTIFICADO

**Síntoma:** 209 jobs duplicados procesados en una sola sesión (2025-12-04 20:*)

**Causas raíz:**
1. ❌ Sistema NO verificaba si los jobs YA estaban en el sheet
2. ❌ Sistema NO verificaba la edad del email
3. ❌ Emails procesados se quedaban en Gmail sin eliminar
4. ❌ Si `processed_bulletins.txt` se borraba → reprocess automático

**Resultado:** 
- Mismo boletín procesado múltiples veces
- Duplicados en Google Sheets
- Emails viejos siendo reprocesados

================================================================================

## ✅ SOLUCIÓN IMPLEMENTADA

### 1. VERIFICACIÓN DE EDAD DEL EMAIL

**Archivo:** `core/automation/job_bulletin_processor.py`

**Función agregada:** `get_email_age_days(message)`

**Lógica:**
```python
# Después de obtener el mensaje
email_age = self.get_email_age_days(message)
if email_age > 7:
    print(f"⏭️  Skipping old email ({email_age} days old)")
    self.save_processed_id(msg_id)  # Marcar para no reprocessar
    continue
```

**Resultado:** Emails >7 días son automáticamente skipped

---

### 2. VERIFICACIÓN DE DUPLICADOS ANTES DE GUARDAR

**Función agregada:** `check_job_exists_in_sheet(job_url, tab_name)`

**Lógica:**
```python
# Para cada job extraído del boletín
for job in jobs:
    job_url = job.get('ApplyURL', '')
    if job_url and self.check_job_exists_in_sheet(job_url, tab_name):
        duplicates_found += 1
    else:
        unique_jobs.append(job)

# Solo guardar jobs únicos
if unique_jobs:
    saved = self.save_to_sheets(unique_jobs, tab_name=tab_name)
```

**Cómo funciona:**
1. Lee TODOS los ApplyURL existentes en el sheet (columna F)
2. Compara cada job extraído del boletín
3. Si la URL YA existe → NO guardar
4. Si la URL es nueva → guardar

**Resultado:** 0 duplicados, sin importar si el boletín se procesa múltiples veces

---

### 3. ELIMINACIÓN AUTOMÁTICA DE EMAILS PROCESADOS

**Funciones agregadas:**
- `mark_email_for_deletion(email_id)`
- `delete_marked_emails()`

**Lógica:**
```python
# Durante el procesamiento
if unique_jobs:
    saved = self.save_to_sheets(unique_jobs, tab_name=tab_name)
    # Marcar email para eliminación
    self.mark_email_for_deletion(msg_id)

# Al final del proceso
self.delete_marked_emails()  # Mueve emails a papelera
```

**Resultado:** 
- Emails procesados se mueven automáticamente a Papelera
- Gmail queda limpio
- No hay confusión sobre qué emails faltan procesar

---

### 4. LISTA DE EMAILS A ELIMINAR

**Variable agregada:** `self.emails_to_delete = []`

**Uso:**
- Se puebla durante el procesamiento
- Se ejecuta al final del proceso
- Se limpia después de eliminar

## 🚨 TROUBLESHOOTING

### Problema: Sigue creando duplicados

**Posibles causas:**
1. ApplyURL no está en columna F
   - **Fix:** Verificar headers en Google Sheets
   - Columna F DEBE ser "ApplyURL"

2. URLs tienen formato diferente
   - **Fix:** Normalizar URLs antes de comparar
   - Agregar `.strip()` y `.lower()`

3. Sheet tiene nombre diferente
   - **Fix:** Verificar que tab_name sea correcto
   - LinkedIn, Indeed, Glassdoor (mayúscula inicial)

---

### Problema: No elimina emails

**Posibles causas:**
1. Permisos de Gmail insuficientes
   - **Fix:** Reautorizar con scopes completos
   - Borrar `token.json` y volver a autenticar

2. Gmail API rate limit
   - **Fix:** Agregar delay entre eliminaciones
   - `time.sleep(0.5)` entre cada trash()

---

### Problema: Todos los emails son "old"

**Posibles causas:**
1. Fecha del sistema incorrecta
   - **Fix:** Verificar fecha/hora en Windows
   - Ajustar timezone si es necesario

2. Emails tienen fecha futura
   - **Fix:** Revisar `parsedate_to_datetime`
   - Agregar validación de fechas razonables
