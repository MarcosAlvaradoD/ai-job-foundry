Key learnings & principles

Never modify working code without strong justification — a core project rule after several regressions caused by refactoring functional components
Subprocess over imports in pipeline: direct subprocess calls to scripts are more reliable than module imports for the daily pipeline, avoiding path/import errors
FIT score parsing must handle multiple formats: "8/10", empty strings, None, and plain integers all appear in Sheets data
Gmail-first over scraping: job alert emails from LinkedIn/Indeed/Glassdoor contain complete job data without rate-limiting risk — process emails before scraping
Bulletin tracking gap: data/state/processed_bulletins.txt tracks email IDs but not dates — old bulletins get reprocessed if the file resets, causing duplicates
OAuth: deleting token.json and re-running full OAuth flow is more reliable than attempting token refresh
PowerShell quirks: use py not python; avoid emoji/Unicode in Write-Host (causes encoding errors); use semicolons not && for command chaining; .\script.bat syntax required
LM Studio vs Ollama IPs: LM Studio uses 172.17.32.1:11434; Ollama uses 127.0.0.1:11434 — not interchangeable
Google Sheets duplicate headers: use get_all_values() not get_all_records() on tabs with duplicate column names (e.g., "Notes")
Playwright verification: Firefox with headless=False and wait_until='domcontentloaded'; 3-second rate limiting between requests; detect expiry in both English and Spanish
